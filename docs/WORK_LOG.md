## 经验教训

### 最大的教训：两个仓库问题
根本原因：缺乏工程纪律
- Cline在不同任务里把代码写到了不同地方
- scriptflow-poc和scriptflow两个仓库同时存在
- 导致部署混乱、代码丢失、时间浪费

正确做法（规则46）：
每次给Cline任务前必须确认：
pwd && git remote -v

### 其他教训
- 发现问题立刻停止，不带着错误继续
- 每个功能必须端到端验证，不只看commit
- 慢下来想清楚再动，才是真正的快
